The Solid Principles Of OOP Design

Single Responsibility Principle (SRP) A class should only have one and only one reason to change. Open-Closed Principle Objects or entities should be open for extension but closed for modification. This means that a class should be extendable without modifying the class itself. Liskov Substitution Principle Let q(x) be a property provable about objects of x of type T. Then q(y) should be provable for objects y of type S where S is a subtype of T.
